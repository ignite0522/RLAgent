# SSRF：常见防御与绕过

## 1. 常见防御

- 限制请求目标为内网 IP（如 127.0.0.1、10.x、172.16.x、192.168.x）
- 禁止 302/重定向或只允许跳转到白名单
- URL 白名单（只允许 http(s) 且域名在白名单）
- 禁用 `file://`、`gopher://`、`dict://` 等协议
- 解析 URL 后校验 host 与请求目标一致（防解析差异）

## 2. 绕过手段

### 内网 IP 表示方式

- **进制/省略**：`127.1`、`0x7f000001`、`2130706433`、`0177.0.0.1`（八进制）
- **DNS 重绑定**：第一次解析到外网，第二次解析到内网
- **localhost 别名**：`localhost`、`127.0.0.1`、`0.0.0.0`
- **IPv6**：`[::1]`、`[::ffff:127.0.0.1]`

### 协议与 URL 解析差异

- **@ 绕过**：`http://evil.com@127.0.0.1`（部分库取 @ 前为 user，后为 host）
- **#  fragment**：`http://127.0.0.1#@evil.com`
- **不同协议**：`file:///etc/passwd`、`gopher://127.0.0.1:6379/...`（打 Redis）、`dict://`
- **URL 编码**：`127.0.0.1` → `%31%32%37%2e%30%2e%30%2e%31` 等，视解析器而定

### 重定向

- 若允许外网：自己的服务器 302 到 `http://127.0.0.1:port`
- 短链接、跳转服务

### 云/本地元数据

- AWS：`http://169.254.169.254/latest/meta-data/`
- 阿里云：`http://100.100.100.200/latest/meta-data/`
- Gopher 打内网 Redis/Memcached 等写 shell 或写 crontab
